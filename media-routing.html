Media Routing / Reten Records / Archives

CyberCrowd needs clear media paths before content becomes a product.

This page defines how CyberCrowd can think about audio, teasers, video, covers, broadcast archives, event media, Reten Records content, and future controlled media areas without hard-wiring production dependencies too early.

Audio

Previews, teasers, full-length tracks, show audio, sound beds, and future Reten Records releases.

Video

Broadcast clips, teasers, room previews, event footage, future VOD, and controlled viewing assets.

Graphics

Covers, posters, creator visuals, product art, room cards, show branding, and promotional material.

Metadata

Titles, credits, pricing notes, access flags, release status, ownership notes, and routing records.

Media routing foundation

Public Preview: crawler-safe teasers, public clips, landing-page media, and discovery-safe content.
Reten Records: music, full-length audio, cover art, release material, catalog content, and future media sales paths.
CyberCrowd Broadcast: live-room assets, camera previews, event teasers, viewer paths, and future archive routing.
CyberShop: product visuals, offer copy, paid access assets, license-related media, and future package presentation.
Protected / RED-card: unreleased files, private records, credentials, internal evidence, legal material, board documents, and anything not safe for public deployment.
This page does not upload, store, sell, stream, or expose media files. It is a foundation map for where media should belong later.

Simple rule

CyberCrowd should know whether a media file is a teaser, public preview, paid item, Reten Records catalog asset, broadcast archive, product graphic, metadata record, or protected file before it is connected to the public web.
