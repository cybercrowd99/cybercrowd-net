CyberCrowd Magic Cursor

CyberCrowd Magic Cursor is the control concept for moving across surfaces in a multi-surface live system. It helps coordinate phones, monitors, cameras, live rooms, show surfaces, media vessels, access states, and archive paths without flattening everything into one screen.

Magic Cursor Purpose

Magic Cursor is not just a pointer. It is a control layer for selecting, guiding, routing, and coordinating activity across CyberCrowd surfaces. It helps operators and trusted users move through live inputs, camera angles, audience moments, media references, and control panels.

What Magic Cursor May Control

  • Phone input surfaces
  • Camera angle selection
  • Monitor-to-monitor coordination
  • Atomic Kricket show surfaces
  • Reten Records media surfaces
  • CyberCrowd public discourse surfaces
  • Live Control Room panels
  • Viewer upgrade states
  • Creator and client controls
  • Archive capture paths
  • WDIG placement decisions

Connection to WDIG

WDIG asks where something belongs. Magic Cursor helps move through those placements and control the surfaces where those things appear. WDIG defines placement. Magic Cursor supports coordination.

Connection to the Live Control Room

The Live Control Room is where the multi-surface system comes together. Magic Cursor can support that control room by moving attention and control between phones, cameras, monitors, show surfaces, media surfaces, viewer access states, and archive paths.

Trust and Surface Discipline

Magic Cursor should operate only within trusted and intentionally defined surfaces. CyberCrowd must avoid uncontrolled movement, unclear custody, accidental exposure, or confusing control paths.

Connection to the CyberCrowd System

Magic Cursor may connect to WDIG Routing, Cam Packages, Viewer Upgrades, Live Control Room, Secure Surface Hardware, Atomic Kricket, Reten Records, public discourse, access passes, and platform archive proof.
