CyberCrowd Access Passes

CyberCrowd Access Passes define how viewers, creators, clients, operators, devices, and future tools may enter controlled parts of the CyberCrowd system.

Access Pass Purpose

CyberCrowd is not an open flat feed. Access must be clear, permissioned, auditable, and connected to what a person, device, viewer, creator, or client is allowed to do.

Possible Access Types

  • Viewer access
  • Creator access
  • Client access
  • Operator access
  • Trial access
  • Camera angle access
  • Live room access
  • Replay or VOD access
  • WDIG Access Pass licensing
  • Future device-bound access

WDIG Access Pass

The WDIG Access Pass is intended as a controlled access model for CyberCrowd tools and device participation. It may use a digital flash-drive key concept: one-time access, device or host binding, non-transferable use, and recovery rules only when verified.

Device-Bound Logic

CyberCrowd access may eventually recognize trusted devices, hosts, kiosks, Surface OS nodes, Magic Cursor surfaces, and other approved control points. Access should not float without custody.

Recovery and Reissue Discipline

Recovery should be limited, verified, and auditable. If a license or access pass is reissued, the old device or host access should be revoked when appropriate.

Connection to Viewer Upgrades

Viewer Upgrades define what a viewer can unlock during an event. Access Passes define the broader permission structure that may support those upgrades, trials, replays, cam packages, and future subscriptions.

Connection to the CyberCrowd System

Access Passes may connect to WDIG Routing, Magic Cursor, Secure Surface Hardware, Cam Packages, Viewer Upgrades, Creator Control, Live Control Room, Archive proof, and future payment systems.

Payment Systems Later

Stripe, subscriptions, x402, or other payment systems should only be added after the access rules, permissions, device logic, and recovery model are clearly defined.
